To make this cassette obey your new hierarchy, you want it to consume your semantic tokens (ThemeTypography) the same way everything else consumes ThemeColorsNotifier.

1) Wire in ThemeTypography using the same notifier pattern

In _ContactHeatmapContent.build, grab both notifiers:

// ...

@override
Widget build(BuildContext context, WidgetRef ref) {
  final colors = ref.watch(themeColorsProvider.notifier);
  final t = ref.watch(themeTypographyProvider); // returns ThemeTypography(colors-notifier)
  // ...

Update the function signature:

Widget _buildStats({

  // ...

Then inside _buildStats, replace the two Text styles (and icon color):

Icon(
  stat.icon,
  size: 14,
  color: colors.content.textSecondary,
),

// value (the number / date range)
Text(
  stat.value,
  style: typography.vizMeta, // tabular + secondary emphasis
),

const SizedBox(height: 2),

// label (“Total messages”, “Archive span”)
Text(
  stat.label,
  style: typography.caption, // tertiary, small, consistent
),

That single swap does a lot:
	•	Keeps “Total messages / Archive span” from reading like headings
	•	Makes the numbers/date range readable but not “hero competing”
	•	Forces the whole stats block into the “viz/meta” lane of your hierarchy

3) Fix the “Open full timeline” button label

Right now it’s const Text(...) so it can’t pick up your tokens. Make it non-const and apply a token:

child: Text(
  'Open full timeline',
  style: t.body.copyWith(
    fontSize: 13, // optional: keep compact in sidebar
    fontWeight: FontWeight.w600,
  ),
),

(If you later add a sidebarAction token, use that instead.)
