## Implemented changes (chronological themes)

### A) Onboarding stepper and labels
- Added explicit `onboardingStarted` state to support true virgin UI.
- Stepper row state now shows all pending before onboarding starts.
- Dev panel and main panel logic aligned to treat pre-start as virgin.
- Added migration-style wording for final migration step:
  - Pending: `Move data to the MessageLens database`
  - Active: `Moving data...`
  - Complete: `All message and contact data loaded into MessageLens`
- Added final ready indicator text:
  - `MessageLens is ready to go!`
  - later changed to hidden until complete.
- Corrected terminal row rendering so complete phase does not show spinner forever.

### B) Timing/pacing
- Removed first-transition bypass so phase pacing applies uniformly.
- Made min phase duration configurable in dev panel (500ms/1.0s/1.5s).
- Persisted min phase duration in overlay DB (`overlay_settings`) using key:
  - `onboarding_min_phase_duration_ms`
- Added overlay settings read/write helpers in overlay database layer.

### C) Dev panel UX/safety
- Prevented deleting `user_overlays.db` from per-row delete action.
- Added present/missing indicator for overlays DB.
- Added pre-run dialog when import/working DB files exist:
  - prompts to delete and continue.

### D) Contacts substage behavior
- Multiple iterations attempted:
  - show pending subtasks instead of hiding
  - paced substage transitions with timers
  - deterministic/ordered substage progression per phase
  - effective completion heuristics (`isComplete`, `current >= total`, `progress >= 1.0`)
  - force transition out of contacts when both contacts subtasks effectively complete
- Contact subtask label updated:
  - `Linking contacts with handles` -> `Organizing your conversations with contacts...`

### E) Logging/debugging
- Added onboarding debug traces, then reduced to transition-only/deduped logs.
- Gated high-volume contact importer debug prints behind constant false.

### F) Import pipeline/rich text extraction
- Rich text importer no longer silently skips when extractor unavailable and candidates exist.
  - now throws explicit `StateError` for missing/unavailable/failing extractor when extraction is required.
- Rust extractor path resolution improved with multiple path candidates (cwd walk-up, bundled path, executable-relative fallback).

### G) Concurrency protection ("single ball" implementation)
- Added new global execution gate provider:
  - `import_execution_gate_provider.dart`
  - tracks owner, holdCount (re-entrant), denied metadata.
- Wired gate into import control view model methods:
  - clear import DB
  - clear working DB
  - start import
  - start migration
  - run import+migration
- Updated chat DB monitor to defer when gate is locked / maintenance active / control processing active.
- Updated monitor to route through control provider `runImportAndMigration()` rather than directly calling services.
